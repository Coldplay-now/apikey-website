Privacy Policy 隐私政策

Last updated: February 9, 2026 最后更新：2026 年 2 月 9 日

TL;DR简而言之

apikey collects zero data. We have no server, no account system, no analytics, and no telemetry. Your API keys are encrypted in the macOS Keychain and never leave your Mac. Period. apikey 不收集任何数据。我们没有服务器、没有账号系统、没有分析工具、没有遥测。你的 API 密钥通过 macOS 钥匙串加密，永远不会离开你的 Mac。就这样。

Data Collection数据收集

apikey does not collect, store, transmit, or share any personal data, usage data, or any other information whatsoever. This includes but is not limited to: apikey 不会收集、存储、传输或共享任何个人数据、使用数据或任何其他信息。包括但不限于：

  • Personal information (name, email, IP address, device identifiers)个人信息（姓名、邮箱、IP 地址、设备标识符）
  • Usage analytics or behavioral data使用分析或行为数据
  • Crash reports or diagnostics崩溃报告或诊断数据
  • API keys or secrets you store in the app你在应用中存储的 API 密钥或密钥值
  • Any metadata about your keys, providers, or usage patterns有关你的密钥、供应商或使用模式的任何元数据

Network Communication网络通信

apikey is a purely local application. It contains no networking code and makes no HTTP requests, WebSocket connections, or any other form of network communication. The app functions entirely offline. apikey 是一款纯本地应用。它不包含任何网络代码，不发起 HTTP 请求、WebSocket 连接或任何其他形式的网络通信。该应用完全离线运行。

The only exception is the optional Key Health Check feature, which — when manually triggered by the user — sends a minimal validation request to the respective API provider (e.g., OpenAI) to verify if a key is still active. This request is sent directly to the provider's API endpoint and does not pass through any of our servers. 唯一的例外是可选的密钥健康检查功能，该功能在用户手动触发时，会向相应的 API 供应商（如 OpenAI）发送一个最小化的验证请求，以确认密钥是否仍然有效。该请求直接发送到供应商的 API 端点，不经过我们的任何服务器。

Data Storage数据存储

API Key ValuesAPI 密钥值

All sensitive key values are stored exclusively in the macOS Keychain, which provides hardware-backed encryption at the operating system level. Key values are never written to disk in plain text. 所有敏感密钥值仅存储在 macOS 钥匙串中，该钥匙串在操作系统层面提供硬件级加密。密钥值永远不会以明文形式写入磁盘。

Metadata元数据

Non-sensitive metadata (key names, provider names, timestamps, key types) is stored in a local SQLite database on your Mac. This data never leaves your device. 非敏感元数据（密钥名称、供应商名称、时间戳、密钥类型）存储在你 Mac 上的本地 SQLite 数据库中。这些数据永远不会离开你的设备。

Clipboard剪贴板

When you copy a key, it is placed on the system clipboard and automatically cleared after a configurable timeout (15–120 seconds). When the app locks, all in-memory key caches are immediately wiped. 当你复制密钥时，它会被放入系统剪贴板，并在可配置的超时时间（15~120 秒）后自动清除。当应用锁定时，所有内存中的密钥缓存会立即清空。

Third-Party Services第三方服务

apikey does not include any third-party SDKs, analytics frameworks, advertising libraries, or tracking tools. There are no third parties involved in the operation of this app. apikey 不包含任何第三方 SDK、分析框架、广告库或跟踪工具。该应用的运行不涉及任何第三方。

apikey leverages the macOS Keychain for secure storage. While we employ best practices to protect your data, we cannot be held responsible for: apikey 利用 macOS 钥匙串进行安全存储。虽然我们采用最佳实践来保护你的数据，但我们不对以下情况承担责任：

  • Loss of API keys due to system failures, hardware damage, or user error因系统故障、硬件损坏或用户操作失误导致的 API 密钥丢失
  • Unauthorized access resulting from compromised system security因系统安全受损而导致的未授权访问
  • Financial or operational losses caused by expired, revoked, or mismanaged keys因密钥过期、被撤销或管理不当造成的经济或运营损失

Users are advised to maintain their own backups and practice good key management hygiene. 建议用户自行维护备份并养成良好的密钥管理习惯。

Changes to This Policy政策变更

If we ever update this privacy policy, changes will be posted on this page with an updated revision date. Given that we collect zero data, we do not anticipate significant changes. 如果我们更新此隐私政策，更改将发布在此页面并更新修订日期。鉴于我们不收集任何数据，我们预计不会有重大变更。
